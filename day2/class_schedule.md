# Day 2：数组基础

## 今天目标

Day 1 学的是“用哈希表记住信息”。Day 2 开始练数组，这是算法题里最常见的数据结构之一。

今天重点不是 JavaScript 数组 API 背诵，而是掌握数组题的三个基本功：

1. 下标：知道 `i` 指向哪里，什么时候越界。
2. 原地修改：在原数组上改，不额外开一个大数组。
3. 写指针：用一个位置专门负责“把有效元素放到哪里”。

学完后你应该能处理这些题型：

- 删除数组中的指定元素。
- 把 0 移动到数组末尾。
- 删除有序数组中的重复项。
- 合并两个有序数组。

## 1. 数组的本质

数组像一排连续编号的储物格：

```text
index:  0   1   2   3
value:  5   8   9   1
```

你可以通过下标直接访问：

```js
const nums = [5, 8, 9, 1];

console.log(nums[0]); // 5
console.log(nums[3]); // 1
```

算法题里经常问的不是“怎么调用某个 API”，而是：

- 当前下标是否合法？
- 当前元素要不要保留？
- 保留的话应该写到哪个位置？
- 写完后有效长度是多少？

## 2. 原地修改是什么

原地修改就是：题目给你一个数组，让你直接在这个数组上改，而不是返回一个全新的数组。

生活例子：

你整理书架时，题目要求“把不想要的书移走，并让剩下的书靠左摆好”。你不能买一个新书架，只能在原书架上挪书。

数组里也一样：

```js
const nums = [3, 2, 2, 3];
```

如果要删除所有 `3`，最终前面的有效区域应该像这样：

```text
[2, 2, _, _]
```

后面的 `_` 不重要，因为题目通常只关心返回的有效长度 `k`，以及 `nums[0]` 到 `nums[k - 1]`。

## 3. 写指针：数组题的核心套路

写指针可以理解成“下一个合格元素应该放的位置”。

比如删除指定元素：

```js
function removeElement(nums, val) {
  let write = 0;

  for (let read = 0; read < nums.length; read++) {
    if (nums[read] !== val) {
      nums[write] = nums[read];
      write++;
    }
  }

  return write;
}
```

解释：

- `read`：从左到右检查每个元素。
- `write`：负责摆放应该保留的元素。
- 遇到不要的元素，`read` 继续走，`write` 不动。
- 遇到要保留的元素，把它写到 `write` 位置，然后 `write++`。

这个套路在很多数组题里都会出现。

## 4. 为什么不要一边遍历一边随便 splice

前端里你可能常用：

```js
nums.splice(i, 1);
```

但算法题里要小心：

1. `splice` 删除元素后，后面的元素会整体前移，容易跳过元素。
2. 每次删除都可能移动很多元素，整体复杂度可能变成 `O(n^2)`。
3. 面试题通常希望你展示指针控制能力。

不是说 `splice` 不能用，而是数组基础训练阶段，我们先练更底层、更稳定的写法。

## 5. 常见数组边界

写数组题时，先问自己：

- 空数组怎么办？
- 只有一个元素怎么办？
- 所有元素都要删除怎么办？
- 没有元素要删除怎么办？
- 题目是否只要求前 `k` 个有效？
- 返回的是数组、长度，还是原地修改后的数组？

这些边界不是细枝末节。很多数组题错，不是思路错，而是下标多走一步或少走一步。

## 6. 今日示例：移动零

问题：给定数组，把所有 `0` 移动到末尾，同时保持非零元素的相对顺序。

示例：

```text
[0, 1, 0, 3, 12] -> [1, 3, 12, 0, 0]
```

思路：

1. 先用写指针把所有非零元素写到前面。
2. 写完后，从 `write` 到数组末尾全部填 `0`。

```js
function moveZeroes(nums) {
  let write = 0;

  for (let read = 0; read < nums.length; read++) {
    if (nums[read] !== 0) {
      nums[write] = nums[read];
      write++;
    }
  }

  for (let i = write; i < nums.length; i++) {
    nums[i] = 0;
  }
}
```

复杂度：

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(1)`，因为只用了几个变量。

## 7. 举一反三

看到这些信号，可以先想“写指针”：

- 删除某些元素。
- 保留符合条件的元素。
- 原地移动元素。
- 返回有效长度。
- 有序数组去重。

前端里的类似场景：

- 从表格数据里过滤掉无效行，但希望复用原数组。
- 对上传队列做状态整理，把可上传项排到前面。
- 把用户勾选的数据按顺序压到前面，后面区域暂时不关心。

### 可以继续优化的点

`moveZeroes` 和 `mergeSortedArrays` 题目主要要求原地修改数组，通常不要求返回数组。你现在返回 `nums` / `nums1` 不影响测试，但在讲题时要说清楚：真正的结果体现在原数组被修改，而不是依赖返回值。

`removeDuplicatesSorted` 也可以从 `read = 1`、`write = 1` 开始写，这样能少处理一次 `read === 0` 的特殊判断。不过你现在的写法更统一，也很适合初学阶段。

### 今天真正学到的东西

数组原地题的核心不是“删除元素”，而是“维护有效区域”：

- `read` 负责看每个元素。
- `write` 负责放下一个有效元素。
- `write` 左边永远是已经处理好的有效区。
- `read` 右边是还没处理的未知区。

这个模型后面会反复出现，尤其是在双指针、滑动窗口、链表题里。
