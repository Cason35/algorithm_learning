# Day 4：双指针

## 今天目标

前 3 天你已经练过哈希表、数组、字符串。Day 4 开始把这些基础组合起来：双指针。

双指针不是一种具体数据结构，而是一种“用两个位置一起推进”的思考方式。它常用于数组、字符串、链表，尤其适合处理：

- 有序数组中的配对问题。
- 字符串回文判断。
- 原地反转。
- 从两端向中间收缩。
- 快慢速度不同的扫描。

今天你要掌握三种常见指针模型：

1. 左右指针：一个在左，一个在右，向中间靠拢。
2. 快慢指针：一个走得快，一个走得慢。
3. 读写指针：Day 2 学过，今天会把它和双指针思路串起来。

## 1. 什么是双指针

想象你和朋友整理一排书。

一种方式是你从左边找，朋友从右边找，两个人一起向中间靠近。这就是左右指针。

另一种方式是你负责快速检查每一本书，朋友只在需要留下书时往前挪一步。这就是快慢/读写指针。

双指针的核心问题是：

```text
哪个指针动？
什么时候动？
为什么这样动不会漏答案？
```

## 2. 左右指针：从两端向中间

适合场景：

- 数组已经有序。
- 题目要求找两个数的关系。
- 题目要求判断回文。
- 题目要求反转数组或字符串。

示例：有序数组两数之和。

```text
nums = [2, 7, 11, 15], target = 9
left = 0, right = 3
```

如果 `nums[left] + nums[right]` 太大，说明右边数字太大，`right--`。

如果和太小，说明左边数字太小，`left++`。

因为数组有序，所以我们知道该移动哪一边。

```js
function twoSumSorted(nums, target) {
  let left = 0;
  let right = nums.length - 1;

  while (left < right) {
    const sum = nums[left] + nums[right];

    if (sum === target) {
      return [left, right];
    }

    if (sum < target) {
      left++;
    } else {
      right--;
    }
  }

  return [-1, -1];
}
```

## 3. 回文里的左右指针

回文判断也天然适合左右指针：

```text
racecar
^     ^
left  right
```

每次比较两端字符：

- 相等：`left++`，`right--`。
- 不相等：不是回文。
- 指针相遇或交错：是回文。

这比先反转整个字符串再比较更省空间。

## 4. 快慢指针

快慢指针常见于：

- 链表中点。
- 链表环检测。
- 原地去重。
- 删除或保留元素。

数组里可以这样理解：

- `fast` 负责从头到尾检查所有元素。
- `slow` 负责记录有效区域边界。

Day 2 的写指针就是快慢指针的一个变体。

## 5. 移动规则比代码更重要

双指针题最容易写错的地方不是语法，而是移动规则。

你写题时要问：

1. 初始位置在哪里？
2. 循环条件是 `left < right` 还是 `left <= right`？
3. 什么情况下移动左指针？
4. 什么情况下移动右指针？
5. 移动一个指针，还是两个都移动？

### 掌握情况

你已经掌握了 Day 4 的主要双指针套路：左右指针从两端收缩、回文比较、原地交换。

四个函数的完成情况：

2. `isPalindromeTwoPointers(s)`：通过。左右两端逐个比较，不需要额外空间。
3. `reverseArrayInPlace(nums)`：通过。原地交换写得很标准，空间复杂度 `O(1)`。
4. `sortedSquares(nums)`：初稿功能样例通过，但用了 `result.unshift(...)`，在 JS 中会把整体复杂度退化到 `O(n^2)`。二次修改后改为预创建结果数组，并用 `write` 从尾部向前写入，复杂度已达标。

### 二次修改记录

`sortedSquares` 的思路已经接近正确：最大平方一定来自最左或最右。问题只在“写入结果”的方式。

修改方向：

```js
const result = new Array(nums.length);
let write = nums.length - 1;
```

每次比较两端绝对值，把较大的平方写到 `result[write]`，然后 `write--`。这样每个位置只写一次，时间复杂度保持 `O(n)`。

### 额外说明

`twoSumSorted` 里负数用例的期望值改成 `[0, 5]` 是正确的，因为 `-3 + 6 = 3`。原脚手架写成 `[1, 5]` 是课程用例错误，已在这次批改里确认。

### 今天真正学到的东西

双指针题不只看“结果对不对”，还要看“指针移动和写入位置是否符合目标复杂度”。

- 左右收缩通常能把双重循环降成一轮扫描。
- 原地交换要保证不会覆盖还没处理的数据。
- 从结果尾部写入，可以避免 `unshift` 这种隐形的整体移动成本。
