## 1. 第一周工具箱

### 哈希表

适合：

- 是否出现过。
- 出现次数。
- 快速配对。
- 按 key 查值。

典型信号：

```text
重复、频次、存在、配对、历史状态
```

### 数组原地操作

适合：

- 删除或保留元素。
- 移动元素。
- 返回有效长度。
- 不允许额外创建大数组。

典型信号：

```text
原地、前 k 个元素、有效区域、保持相对顺序
```

### 字符串扫描

适合：

- 规范化输入。
- 连续字符统计。
- 字符比较。
- 字符串构造。

典型信号：

```text
大小写、标点、字符、单词、回文、压缩
```

### 双指针

适合：

- 从两端比较。
- 有序数组配对。
- 原地反转。
- 回文判断。

典型信号：

```text
有序、两端、反转、回文、左右收缩
```

### 滑动窗口

适合：

- 连续子串或连续子数组。
- 最长、最短、固定长度。
- 窗口内维护和、集合、频次。

典型信号：

```text
连续、最长、最短、子串、子数组、窗口
```

### 前缀和

适合：

- 区间和。
- 多次查询。
- 子数组和等于某个值。
- 需要统计历史累计状态。

典型信号：

```text
区间和、子数组和、累计、从 i 到 j
```

## 2. 怎么判断用哪个算法

看到题目时，先问 5 个问题：

1. 输入是数组还是字符串？
2. 题目是否强调“连续”？
3. 是否需要频次或是否出现过？
4. 是否有“有序”或“两端”的条件？
5. 是否有很多区间和或累计和？

对应关系：

- “频次 / 出现过”优先想哈希表。
- “连续 + 最长/最短”优先想滑动窗口。
- “连续 + 和”优先想前缀和或滑动窗口。
- “有序 + 两数关系”优先想双指针。
- “原地修改”优先想读写指针。

## 3. 今日综合题思路

今天作业有 4 题：

1. 稳定去重：哈希表 + 数组。
2. 清洗后回文：字符串扫描 + 双指针。
3. 最多 k 种字符的最长子串：滑动窗口 + Map。
4. 0 和 1 数量相等的子数组数量：前缀和 + Map。

第 4 题有个转换：

```text
0 看作 -1
1 看作 +1
```

这样“0 和 1 数量相等”就变成“子数组和等于 0”。然后就可以用 Day 6 的前缀和 + Map。
