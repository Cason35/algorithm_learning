## 1. 字符串和数组有什么不同

字符串可以像数组一样用下标读取：

```js
const word = "hello";

console.log(word[0]); // h
console.log(word[4]); // o
```

但字符串不能像数组那样原地修改：

```js
const word = "hello";
word[0] = "H";

console.log(word); // 仍然是 hello
```

因为 JS 字符串是不可变的。你想“修改字符串”，本质上是创建一个新字符串。

生活例子：

数组像一排可以挪动的积木，Day 2 里我们可以把元素挪来挪去。

字符串像已经打印出来的一张纸，你不能直接改纸上的一个字，只能重新打印一张改过的新纸。

## 2. 字符串常用操作

常用 API：

```js
const s = "Hello World";

console.log(s.length); // 11
console.log(s.toLowerCase()); // hello world
console.log(s.includes("World")); // true
console.log(s.slice(0, 5)); // Hello
```

算法题里最常用的是：

- `s[i]`：读取字符。
- `s.length`：控制边界。
- `toLowerCase()`：统一大小写。
- `slice(start, end)`：截取子串。
- `for...of`：按字符遍历。

## 3. 字符串扫描

字符串扫描就是从左到右看每个字符，并维护一点状态。

比如统计连续字符：

```text
aaabbc
```

从左到右看：

- 看到 `a`，连续次数是 1。
- 下一个还是 `a`，次数变 2。
- 下一个还是 `a`，次数变 3。
- 下一个变成 `b`，说明 `a` 这一段结束，记录 `a3`。

这类题的关键是：当前字符和上一个字符是否相同。

## 4. 回文字符串

回文就是正着读和反着读一样：

```text
level
racecar
上海自来水来自海上
```

如果题目要求忽略大小写和标点，比如：

```text
A man, a plan, a canal: Panama
```

我们先把它清洗成：

```text
amanaplanacanalpanama
```

再判断是否回文。

清洗字符串时可以先判断字符是否是字母或数字：

```js
function isAlphaNumeric(char) {
  return /^[a-z0-9]$/i.test(char);
}
```

今天可以先用这种直观写法，后面双指针专题会再讲更省空间的版本。

## 5. 公共前缀

公共前缀就是一组字符串开头共同拥有的部分。

```text
flower
flow
flight
```

它们的最长公共前缀是：

```text
fl
```

思路可以很朴素：

1. 先拿第一个字符串当候选前缀。
2. 去和后面的每个字符串比较。
3. 只要某个字符串不是以这个前缀开头，就把前缀缩短一位。
4. 前缀缩到空字符串时，直接返回 `""`。

### 可以继续优化的点

`reverseString` 里反复使用 `result += s[i]` 很直观，但在字符串很长时，字符串不可变会带来额外拷贝。更稳的写法是先 `push` 到数组里，最后 `join("")`。

`longestCommonPrefix` 可以在 `prefix === ""` 时提前返回，避免后续无意义比较：

```js
if (prefix === "") return "";
```

`isValidPalindrome` 目前只按英文大小写字母和数字处理，这是这类基础题最常见的范围。以后遇到 Unicode 字符，比如中文、emoji、带重音字母时，判断规则要重新定义。

### 今天真正学到的东西

字符串题的第一步往往不是写循环，而是先确认“字符规则”：

- 是否区分大小写？
- 是否保留空格和标点？
- 是否只处理英文和数字？
- 是否需要返回新字符串，还是只返回判断结果？

确认规则之后，再决定用清洗、扫描、双指针或计数。这个思路比记某个 API 更重要。
