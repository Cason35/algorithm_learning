# Day 12: 排序基础

## 今天目标

Day 12 学排序。排序不是单纯把数组变整齐，它经常是很多题的“预处理步骤”：

- 想找重复：排序后相同元素会挨在一起。
- 想用双指针：很多双指针题要求数组先有序。
- 想合并区间：通常先按起点排序。
- 想做 Top K：排序可以先给你一个最直观的基准解。

今天你要掌握 4 件事：

1. JS `sort` 默认是按字符串排序，不是数字排序。
2. 合并两个有序数组，是归并排序的基础动作。
3. 归并排序的核心是“先拆小，再合并”。
4. 快速排序的核心是“选基准，再分区”。

## 1. 生活例子

想象你在整理一叠发票。

如果你按金额从小到大排：

```text
12, 3, 25, 100
```

人脑会觉得应该变成：

```text
3, 12, 25, 100
```

但 JS 的 `sort()` 默认会先把元素当字符串比较：

```js
[12, 3, 25, 100].sort(); // [100, 12, 25, 3]
```

因为字符串比较时，`"100"` 的第一个字符 `"1"` 比 `"3"` 小。

所以数字排序要写比较函数：

```js
nums.sort((a, b) => a - b);
```

含义是：

- 返回负数：`a` 排在 `b` 前面。
- 返回正数：`a` 排在 `b` 后面。
- 返回 0：相对顺序不变或无所谓。

## 2. JS sort 的注意点

### 默认排序会踩坑

```js
[1, 10, 2].sort(); // [1, 10, 2]
```

数字升序应该这样写：

```js
[1, 10, 2].sort((a, b) => a - b); // [1, 2, 10]
```

数字降序：

```js
[1, 10, 2].sort((a, b) => b - a); // [10, 2, 1]
```

### sort 会修改原数组

```js
const nums = [3, 1, 2];
const sorted = nums.sort((a, b) => a - b);

console.log(nums);   // [1, 2, 3]
console.log(sorted); // [1, 2, 3]
```

如果题目要求不要修改输入数组，要先复制：

```js
const sorted = [...nums].sort((a, b) => a - b);
```

## 3. 合并两个有序数组

假设有两队人都已经按身高排好：

```text
A: 1, 4, 7
B: 2, 3, 9
```

你不需要重新乱排，只要每次看两队最前面谁更小，把谁拿出来：

```text
1 -> 2 -> 3 -> 4 -> 7 -> 9
```

代码思路：

```js
function mergeSortedArrays(nums1, nums2) {
  const result = [];
  let i = 0;
  let j = 0;

  while (i < nums1.length && j < nums2.length) {
    if (nums1[i] <= nums2[j]) {
      result.push(nums1[i]);
      i += 1;
    } else {
      result.push(nums2[j]);
      j += 1;
    }
  }

  while (i < nums1.length) {
    result.push(nums1[i]);
    i += 1;
  }

  while (j < nums2.length) {
    result.push(nums2[j]);
    j += 1;
  }

  return result;
}
```

复杂度：

- 时间：`O(n + m)`
- 空间：`O(n + m)`

## 4. 归并排序

归并排序像整理两摞已经排好的牌。

它分两步：

1. 拆：把数组不断拆成左右两半，直到每组只剩 1 个元素。
2. 合：把两个有序小数组合并成一个更大的有序数组。

例如：

```text
[5, 2, 4, 1]

拆：
[5, 2] [4, 1]
[5] [2] [4] [1]

合：
[2, 5] [1, 4]
[1, 2, 4, 5]
```

它的稳定复杂度是：

- 时间：`O(n log n)`
- 空间：`O(n)`

## 5. 快速排序

快速排序像选一个“中间参考价”，然后把商品分成两边：

- 比它便宜的放左边。
- 比它贵的放右边。
- 最后递归处理左右两边。

例如：

```text
[5, 2, 4, 1, 3]
pivot = 3

小于 3: [2, 1]
等于 3: [3]
大于 3: [5, 4]
```

再分别排序左右两边，最后拼起来。

复杂度：

- 平均时间：`O(n log n)`
- 最坏时间：`O(n^2)`
- 空间：取决于写法，递归通常会用到调用栈

## 6. 举一反三

遇到排序题时，先问自己：

1. 排序后，相邻元素之间会不会产生有用关系？
2. 排序后，能不能使用双指针？
3. 题目要求是否允许修改原数组？
4. 比较规则是不是单纯数字升序，还是多条件排序？

多条件排序例子：

```js
people.sort((a, b) => {
  if (a.age !== b.age) {
    return a.age - b.age;
  }

  return a.name.localeCompare(b.name);
});
```

含义是：先按年龄升序；年龄相同，再按名字排序。
