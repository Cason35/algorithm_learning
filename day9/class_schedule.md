# Day 9：队列

## 今天目标

Day 8 学了栈，规则是“后进先出”。Day 9 学队列，规则刚好反过来：

```text
先进先出
```

也就是先排队的人，先被处理。

JavaScript 数组可以用 `push` 入队，但不建议频繁用 `shift` 出队，因为 `shift()` 会让后面的元素整体前移，单次可能是 `O(n)`。

今天我们用更稳的写法：

```js
const queue = [];
let head = 0;

queue.push(value); // 入队
const front = queue[head]; // 队头
head++; // 出队
```

## 1. 生活例子

队列就像排队买奶茶：

- 先到的人站在前面。
- 店员每次服务队头。
- 新来的人排到队尾。

这就是队列：

- 入队：放到队尾。
- 出队：从队头拿走。
- 队头：下一个要被处理的人。

## 2. 队列适合什么题

看到这些信号，可以想队列：

- 按到达顺序处理。
- 请求、消息、任务排队。
- 先进先出。
- 最近一段时间内的请求。
- 层序遍历。
- 广度优先搜索。

前端里的类似场景：

- 消息队列。
- 任务调度。
- 请求限流。
- Toast 依次展示。
- 页面埋点按顺序上报。

## 3. 为什么少用 shift

`shift()` 看起来像真正的出队：

```js
const first = queue.shift();
```

但它会把数组后面的所有元素向前挪一位。数据小时没事，算法题里如果大量出队，就可能退化。

更推荐：

```js
let head = 0;

const first = queue[head];
head++;
```

数组内容没有真的删除，但 `head` 左边的区域已经逻辑失效。

## 4. 固定时间窗口队列

例题：统计最近 `3000ms` 内有多少请求。

每来一个请求时间 `t`：

1. 把 `t` 入队。
2. 队头如果小于 `t - 3000`，说明过期，移动 `head`。
3. 当前有效请求数是 `queue.length - head`。

这类题的本质是：队列保存“还在有效时间范围内的请求”。

## 5. 固定大小队列

移动平均值也适合队列。

比如窗口大小为 3，数据流来了：

```text
1, 10, 3, 5
```

每次新值入队，同时维护 `sum`：

- 队列长度超过 3 时，队头出队，并从 `sum` 里减掉。
- 当前平均值是 `sum / 有效元素数量`。

如果每次都重新求和，复杂度会变高。维护 `sum` 是队列题常见的配套状态。

## 6. 循环队列直觉

循环队列是固定容量队列的一种实现方式。

它用数组加两个指针：

- `front`：队头位置。
- `size`：当前元素数量。
- 入队位置可以通过 `(front + size) % capacity` 算出来。

取模的意义是：到数组尾部以后绕回开头。

今天不要求写完整类，只要能理解这种“固定空间复用”的思想。

## 7. 双端队列与单调队列

双端队列可以从两头进出：

- 队头出。
- 队尾进。
- 队尾也能出。

滑动窗口最大值会用到单调队列：

- 队列里存下标。
- 队头是当前窗口最大值下标。
- 新元素进来时，把队尾所有比它小的下标弹掉。
- 窗口左边界移动时，把过期下标从队头移走。

这其实和 Day 8 的单调栈很像，只是队列还要处理“窗口过期”。
