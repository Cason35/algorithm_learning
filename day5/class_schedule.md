# Day 5：滑动窗口

## 今天目标

Day 4 学了双指针。Day 5 进入一个非常高频的双指针分支：滑动窗口。

滑动窗口可以理解成数组或字符串上的一个连续区间：

```text
[ left ... right ]
```

我们一边移动 `right` 扩大窗口，一边在条件不满足或已经满足时移动 `left` 缩小窗口。

今天你要掌握 4 件事：

1. 固定长度窗口：窗口大小不变。
2. 可变长度窗口：窗口大小会根据条件扩大或缩小。
3. 窗口状态：窗口里当前有什么，比如和、字符次数、去重集合。
4. 什么时候更新答案：扩张时更新，还是收缩后更新。

## 1. 生活例子

想象你在看一段监控录像，屏幕上每次只显示连续 10 秒。

你不是每次都重新打开一段新录像，而是：

- 右边进入 1 秒新画面。
- 左边丢掉 1 秒旧画面。
- 中间这 10 秒就是当前窗口。

这就是固定窗口。

如果你在找“最短的一段录像，里面至少出现 3 次某个事件”，窗口长度就不是固定的：

- 事件还不够，右边继续扩张。
- 事件够了，左边尝试收缩。

这就是可变窗口。

## 2. 固定窗口

例题：长度为 `k` 的连续子数组最大和。

暴力做法会重复计算：

```text
[1, 2, 3] 求和
   [2, 3, 4] 求和
```

其实两个窗口大部分内容是重叠的。窗口向右移动时：

- 加入右边新元素。
- 移除左边旧元素。

```js
function maxSumFixedWindow(nums, k) {
  if (k <= 0 || k > nums.length) return 0;

  let windowSum = 0;

  for (let i = 0; i < k; i++) {
    windowSum += nums[i];
  }

  let best = windowSum;

  for (let right = k; right < nums.length; right++) {
    windowSum += nums[right];
    windowSum -= nums[right - k];
    best = Math.max(best, windowSum);
  }

  return best;
}
```

## 3. 可变窗口

例题：不含重复字符的最长子串。

窗口状态可以用 `Set` 保存当前窗口里有哪些字符：

1. `right` 尝试加入新字符。
2. 如果这个字符已经在窗口里，就移动 `left`，直到重复字符被移走。
3. 每次窗口合法后，更新最长长度。

核心不是“遇到重复就清空”，而是只移动到刚好合法。

## 4. 最短窗口

有些题不是找最长，而是找最短。

例题：正整数数组里，找和至少为 `target` 的最短连续子数组长度。

思路：

- `right` 不断扩张，让窗口和变大。
- 当窗口和已经大于等于 `target`，说明当前窗口有效。
- 这时移动 `left` 尝试缩短窗口。
- 每次有效时更新最小长度。

这里的关键是：只有窗口已经满足条件，才有资格收缩。

## 5. 字符频次窗口

异位词题经常用字符频次窗口。

比如在 `s = "cbaebabacd"` 里找 `p = "abc"` 的异位词起点。

窗口长度固定为 `p.length`，只要窗口内每个字符次数和 `p` 一样，就是一个答案。

今天作业里可以用 `Map` 写，先追求清晰。后面如果只处理小写英文，可以用长度为 26 的数组优化。

## 6. 滑动窗口解题模板

```js
let left = 0;

for (let right = 0; right < input.length; right++) {
  // 1. 把 input[right] 加入窗口

  while (/* 窗口需要收缩 */) {
    // 2. 把 input[left] 移出窗口
    left++;
  }

  // 3. 根据当前窗口更新答案
}
```

- 窗口里维护什么？
- 什么时候扩大？
- 什么时候缩小？
- 答案在扩大后更新，还是收缩时更新？

### 需要二次修改的点

`maxSumFixedWindow` 应该维护一个持续更新的 `windowSum`：

```js
windowSum += nums[right];
windowSum -= nums[right - k];
```

这样窗口每次右移只做一次加法和一次减法，时间复杂度是 `O(n)`。

`findAnagramIndices` 应该维护固定长度窗口的字符频次。因为题目限定小写英文字母，可以用两个长度为 26 的数组，或者一个差异数组：

```js
const base = "a".charCodeAt(0);
const diff = new Array(26).fill(0);
```

窗口右边加入字符时更新一次，左边移出字符时再更新一次。不要每次重新 `slice` 整个窗口。

### 今天真正学到的东西

滑动窗口的关键不是“能移动左右指针”，而是“窗口状态要复用”。

- 固定窗口：旧元素出去，新元素进来，窗口和增量更新。
- 可变窗口：条件不满足就扩张，条件满足或冲突时收缩。
- 频次窗口：字符进入和离开时只更新对应字符的计数。

如果每次窗口移动后都重新扫描整个窗口，就还没有真正用上滑动窗口的优势。
