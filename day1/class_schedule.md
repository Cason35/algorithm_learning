# Day 1：复杂度 + 哈希表

## 今天目标

今天不急着刷很多题，先建立两个底层直觉：

1. 复杂度：判断一段代码会不会随着数据变大而变慢。
2. 哈希表：用 `Map` / `Set` 让“查找是否出现过”这类问题从慢变快。

学完后你应该能看懂并写出这几类题：

- 判断数组里有没有重复元素。
- 统计字符串中每个字符出现次数。
- 判断两个字符串是不是异位词。
- 在数组中找两个数相加等于目标值。

## 1. 什么是算法复杂度

算法可以理解成“解决问题的步骤”。复杂度就是问一句：当输入数据变大时，这套步骤会膨胀成什么样？

生活例子：

- 你在一排 100 个快递里找自己的快递，一个一个看，这是 `O(n)`。
- 如果快递柜按取件码建了索引，你输入取件码直接打开，这是接近 `O(1)`。
- 如果你拿每个快递都和另外每个快递比一遍，就是 `O(n^2)`，数据一大就会很痛。

前端里也能看到复杂度：

- 页面上 10000 条列表数据，如果每次搜索都从头扫一遍，是 `O(n)`。
- 如果先按 `id` 建一个 `Map`，之后通过 `id` 查详情，就是 `O(1)`。

常见复杂度从好到坏：

```text
O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n)
```

今天先重点掌握 `O(1)`、`O(n)`、`O(n^2)`。

## 2. 哈希表是什么

哈希表的直觉像一本“按关键词查的通讯录”。

不用哈希表时，你要问“张三在不在这个名单里”，可能要从第一行扫到最后一行。

用哈希表时，你把名字当作 key 存起来：

```js
const seen = new Set();

seen.add("张三");
seen.add("李四");

console.log(seen.has("张三")); // true
```

`Set` 适合回答：这个东西有没有出现过？

`Map` 适合回答：这个 key 对应的值是什么？

```js
const countMap = new Map();

for (const char of "banana") {
  countMap.set(char, (countMap.get(char) || 0) + 1);
}

console.log(countMap); // b:1, a:3, n:2
```

## 3. Map 和 Object 怎么选

在算法题里，我建议你优先用：

- `Set`：只关心“有没有”。
- `Map`：关心“key 对应的值”。

`Object` 也能做映射，但它的 key 会被转成字符串，还可能碰到原型链上的属性。算法学习阶段，用 `Map` 更干净。

## 4. 举一反三：看到这些关键词就想到哈希表

题目里如果出现这些信号，可以先想 `Map` / `Set`：

- “是否存在”
- “是否重复”
- “出现次数”
- “配对”
- “两个集合有没有交集”
- “按 id 快速查找”

前端工作中的对应场景：

- 接口返回用户列表，要根据 `userId` 快速找用户信息。
- 埋点事件要去重，避免同一事件重复上报。
- 搜索框输入后，要判断缓存里有没有这个关键词的结果。
- 多个接口返回的数据要按 `id` 合并。

## 5. 今日示例：找第一个重复元素

问题：给一个数组，返回第一个重复出现的数字；如果没有重复，返回 `null`。

思路：

1. 准备一个 `Set`，叫 `seen`。
2. 从左到右扫描数组。
3. 如果当前数字已经在 `seen` 里，说明它重复了，直接返回。
4. 如果没出现过，就加入 `seen`。

```js
function firstDuplicate(nums) {
  const seen = new Set();

  for (const num of nums) {
    if (seen.has(num)) {
      return num;
    }

    seen.add(num);
  }

  return null;
}

console.log(firstDuplicate([3, 1, 4, 1, 5])); // 1
console.log(firstDuplicate([1, 2, 3])); // null
```

复杂度：

- 时间复杂度：`O(n)`，因为数组只扫一遍。
- 空间复杂度：`O(n)`，最坏情况下所有数字都要放进 `Set`。
